# Data Dictionary

All tables are synthetic and generated by `scripts/build_safety_reporting_quality_lab.py`.

## `data/field_locations.csv`

- `location_id`: Synthetic location key.
- `region`: Operating region.
- `division`: Business division represented in the synthetic field network.
- `service_branch`: Human-readable branch label.
- `employee_partners`: Synthetic employee-partner count used for training assignment sizing.
- `support_tier`: `standard` or `high_touch`, used to model locations needing more reporting support.
- `safety_lead_tenure_years`: Synthetic tenure of the local safety lead.
- `go_live_wave`: Report rollout wave, 1 through 4.

## `data/safety_report_runs.csv`

- `run_id`: Synthetic report run key.
- `run_date`: Date the report was scheduled.
- `report_name`: Daily, weekly, monthly, or ad hoc report type.
- `cadence`: Report cadence.
- `source_system`: Synthetic source system family such as Safety Logic, MSDS, ISN, Browz, PICS, or Corporate Request.
- `location_id`: Synthetic location key.
- `region`: Operating region.
- `division`: Business division.
- `support_tier`: Location support tier at the time of the run.
- `sla_hours`: Expected completion window for the report run.
- `refresh_delay_hours`: Simulated hours between scheduled refresh and completed refresh.
- `sla_met`: Whether `refresh_delay_hours` was within `sla_hours`.
- `records_expected`: Expected input record count.
- `completeness_rate`: Simulated share of expected records present and usable.
- `duplicate_keys`: Count of duplicate incident or report keys detected.
- `unmapped_records`: Count of records missing a clean employee-partner, location, or owner mapping.
- `completed_at`: Timestamp when the synthetic report refresh completed.

## `data/safety_report_exceptions.csv`

- `exception_id`: Synthetic exception key.
- `run_id`: Report run that created the exception.
- `run_date`: Report date.
- `report_name`: Report that surfaced the exception.
- `source_system`: Source system family.
- `location_id`: Synthetic location key.
- `region`: Operating region.
- `division`: Business division.
- `exception_type`: Exception category.
- `severity`: Critical, high, medium, or low.
- `owner_group`: Group expected to resolve or coordinate the exception.
- `age_days`: Age of the exception at analysis time.
- `status`: Open or closed.
- `resolution_hours`: Filled only for closed exceptions.
- `requires_field_followup`: Whether the exception likely needs field communication or coaching.

## `data/report_rollout_training.csv`

- `assignment_id`: Synthetic training assignment key.
- `location_id`: Synthetic location key.
- `region`: Operating region.
- `division`: Business division.
- `go_live_wave`: Report rollout wave.
- `module_name`: Training module.
- `audience`: Intended audience for the training.
- `assigned_employee_partners`: Number assigned to the module.
- `completed_employee_partners`: Number who completed the module.
- `overdue_employee_partners`: Number past expected completion.
- `completion_rate`: Completed divided by assigned.

## `data/data_quality_checks.csv`

- `check_id`: Synthetic quality check key.
- `run_id`: Report run being tested.
- `run_date`: Report date.
- `report_name`: Report under test.
- `source_system`: Source system family.
- `location_id`: Synthetic location key.
- `check_name`: Data quality rule name.
- `records_tested`: Number of rows tested by the rule.
- `failed_records`: Number of rows that failed the rule.
- `pass_rate`: Passing records divided by tested records.

## `data/field_support_requests.csv`

- `request_id`: Synthetic support request key.
- `request_date`: Date submitted.
- `location_id`: Synthetic location key.
- `region`: Operating region.
- `division`: Business division.
- `request_type`: Support request category.
- `priority`: Urgent, standard, or low.
- `target_hours`: Service target based on priority.
- `turnaround_hours`: Simulated time to resolve or respond.
- `met_target`: Whether the target was met.
- `requester_group`: Corporate Safety, Field Operations, or Business Leader.

## Generated Outputs

- `analysis/outputs/headline_metrics.json` and `.csv`: Summary metrics for the full run.
- `analysis/outputs/report_sla_quality_summary.csv`: SLA, refresh, completeness, duplicate, and unmapped-record summary by report.
- `analysis/outputs/data_quality_rule_summary.csv`: Failed-record rates by source system and quality rule.
- `analysis/outputs/field_exception_action_queue.csv`: Top open exception groups ranked by severity, age, volume, and follow-up need.
- `analysis/outputs/rollout_training_adoption_summary.csv`: Training completion by region, division, and rollout wave.
- `analysis/outputs/monthly_reporting_reliability.csv`: Monthly trend inputs used in the rendered SLA/completeness evidence.
- `analysis/outputs/field_support_request_sla_summary.csv`: Support turnaround performance by request type and priority.
